# Local proving deployment profile

Proof generation occurs in the holder-controlled environment. Credentials and witnesses remain local; the verifier receives only the presentation and protocol metadata.

Controls include protected local storage, trusted runtime updates, request authentication, transcript binding, anti-exfiltration review, resource limits and recovery. The deployment evidence records supported devices, protected-key assumptions and failure behavior when local proving is unavailable.
