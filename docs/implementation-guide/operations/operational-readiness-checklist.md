# Operational readiness checklist

- [ ] On-call and decision authorities are named.
- [ ] Policy and registry changes require authenticated approval.
- [ ] Revocation propagation is measured.
- [ ] Key rotation and restoration have been exercised.
- [ ] Stale, conflicting and unavailable state have tested outcomes.
- [ ] Evidence preservation excludes unnecessary proof content.
- [ ] Redress and correction routes are operational.
- [ ] Degraded modes and stop authority are approved.
