# Policy update runbook

Trigger: approved semantic or relying-party policy change. Validate authority, version the policy, run regression and negative tests, stage deployment, verify decision receipts, monitor rejection changes and retain rollback capability. Closure requires evidence that all active verifiers use the intended version.
