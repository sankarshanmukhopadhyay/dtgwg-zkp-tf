# Contributing

Before opening a pull request:

- use stable identifiers;
- state normative status;
- name adversary and horizon for privacy claims;
- state non-claims;
- map content to owners and ADRs;
- avoid selecting cryptography without a profile requirement;
- run the documentation validator.
