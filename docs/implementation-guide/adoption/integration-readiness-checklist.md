# Integration readiness checklist

- [ ] Context and predicate identifiers are stable.
- [ ] Issuer and registry authority are documented.
- [ ] Assurance and disclosure boundaries are approved.
- [ ] Request and presentation transcripts are canonical.
- [ ] Status, expiry and stale-state behavior are deterministic.
- [ ] Error codes are mapped to operator actions.
- [ ] Holder and delegated-agent authority are separated.
- [ ] Logs exclude unnecessary credential and proof content.
- [ ] Test fixtures can be exchanged independently.
- [ ] Incident, rollback and redress owners are named.
