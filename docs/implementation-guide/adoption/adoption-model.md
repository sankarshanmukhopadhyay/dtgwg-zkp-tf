# Adoption model

## Governance dimensions

Every stage records authority, delegated operators, scope, enforcement points, revocation paths, evidence owners and residual-risk authority. Advancement is a governance decision supported by evidence, not a function of elapsed time.

## Stage gates

### A — Exploration
Entry requires a named problem, candidate predicate and accountable sponsor. Exit requires an assurance-boundary hypothesis, disclosure-boundary hypothesis, threat discovery and explicit prohibition on production reliance.

### B — Controlled prototype
Exit requires minimum viable issuer, holder, verifier and status behavior; synthetic or non-sensitive data; deterministic failure codes; and documented assumptions.

### C — Interoperability pilot
Exit requires two independently controlled implementations, portable fixtures, registry and revocation tests, operational ownership and traceable conformance evidence.

### D — Limited production
Exit requires bounded users and relying parties, production key custody, deployment threat assessment, incident and redress procedures, monitored fallback behavior and approved residual risk.

### E — General production
Entry requires independent assessment, service objectives, tested recovery, algorithm and policy migration plans, supply-chain controls and scheduled conformance re-evaluation.
