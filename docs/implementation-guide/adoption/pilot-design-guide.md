# Pilot design guide

A pilot should answer a bounded interoperability or assurance question. It should not simulate production by weakening production controls.

## Required pilot packet

- participating authorities and operators;
- profiles and contexts in scope;
- synthetic or approved data policy;
- component and version inventory;
- shared fixture set;
- positive and negative scenarios;
- registry and status availability assumptions;
- evidence collection plan;
- incident and stop authority;
- success, failure and exit criteria.

At least one participant should be independently controlled so the pilot tests interoperability rather than a single implementation stack.
