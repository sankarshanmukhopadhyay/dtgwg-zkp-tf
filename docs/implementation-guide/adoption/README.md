# Adoption pathway

The adoption programme separates experimentation from production reliance. Each stage has evidence-based entry and exit gates.

| Stage | Intended outcome | Minimum evidence |
|---|---|---|
| A — Exploration | Validate predicate and authority assumptions | Context record, threat hypothesis, prototype boundary |
| B — Controlled prototype | Demonstrate a bounded end-to-end flow | Synthetic fixtures, negative tests, named owners |
| C — Interoperability pilot | Prove portability across independent components | Plugfest report, conformance evidence, status tests |
| D — Limited production | Operate for a bounded relying-party population | Production controls, incident runbooks, residual-risk approval |
| E — General production | Sustain independently assessable operation | Independent assessment, recovery evidence, periodic re-evaluation |
