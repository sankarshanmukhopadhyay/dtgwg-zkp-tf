# Assurance boundary template

| Field | Required content |
|---|---|
| Boundary ID | Stable `AB-xxx` identifier |
| Predicate/profile | Applicable predicate and profile identifiers |
| Statement established | Exact verifier-reliance proposition |
| Negative meaning | What the statement does not establish |
| Against whom | Adversary and collusion assumptions relevant to reliance |
| For how long | Validity, status freshness, policy lifetime and assurance horizon |
| Alongside what | Required credential, registry, protocol and operational evidence |
| Upstream determination | Determination made outside the proof |
| Governance dependency | Policy, accreditation and recognition framework |
| Accountable party | Party responsible for correctness and correction |
| Failure modes | Conditions under which reliance becomes invalid |
| Redress | Contest, correction and propagation route |
| Threat references | Applicable `THR-xxx` identifiers |
| Conformance evidence | Test, audit and fixture references |

An assurance boundary must not state or imply that proof validity establishes biometric correctness, civil identity, global uniqueness, non-coercion, agent authority or transaction authorization unless a separate, explicit proposition and evidence path establishes that claim.
